Scroll-Wiz

Intro

Scroll-Wiz is a Javascript micro library, created as a solution to facilitate the implentation of web scroll-controlled animated elements (front-end webdev). While scroll-wiz script triggers the actions based on scroll values, the animation's implentation mainly happens in your css code.

Each scroll-controlled element takes a minimum of 2 CSS classes:

  • A unique Main Class (for the out-of-range scroll-action properties). This class should be added to the element in HTML. Alternatively you can use a tag selector, in case it's a unique tag, a header for instance.
  • And an Action Class (for the in-range scroll-action properties). This class will be added automatically to the HTML when the scroll action is in-range.

The main class should have the properties for the element outside of the action range. For example, if the result for a specific element is meant to take place between scroll values (30 - 300), the main class should have the properties that will apply to the element before reaching scroll value of 30 pixels, and after scrolling past 300 pixels. It should -as well- include the transition or keyframes animation properties. This class should be added to the element in html.
The action class, which includes the properties that the element will have within the action range. You need to declare this class in the CSS file, but no need to add it to the element in html.
Each action class can be used several times, at several action ranges, or you can use a separate action class for each animation.

Javascript code in the file (scrollwiz/sw.js) controls adding and removing classes at a specific vertical scroll value, and keeps track of the scroll-state.

Click here for repo on Github.

Important Notes

Each element's main class should be a unique name. You might be tempted to use an ID instead of a class, but this will create a specificity issue when changing the animated properties values in the animation class, if you do use an ID, you need to add an !important flag for the properties in the animation class, which I strongly advice against. On the other hand, if you use the same main class for several elements, that you wish to animate, only the first element will be captured when you assign to a variable in Javascript using
const element = document.querySelector('.element-main-class')
